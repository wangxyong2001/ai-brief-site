| 系统 | AI Daily Brief (ai.tomabc.com) |
| 事故类型 | 功能失效 |
| 影响范围 | 全站用户 |
| 严重程度 | 高 |

---

## 事故描述

### 问题现象

用户在使用 AI Daily Brief 系统时发现以下功能异常：

1. **分类标签点击后内容为空**
   - 用户点击任意分类标签后，页面显示无内容
   - 前端路由跳转正常，但数据加载失败
   - 错误原因：数据源与标签不匹配

2. **详情页无翻译内容**
   - Brief 详情页无法显示对应的翻译文章
   - 页面布局正常，但核心内容缺失
   - 错误原因：Brief 与 Articles 表关联断裂

3. **前端 ID 计算错误**
   - 前端使用 `index + 1` 计算文章 ID
   - 该值不等于数据库实际 ID
   - 导致数据查询失败

### 影响评估

- 全站用户无法正常浏览分类内容
- Brief 详情页核心功能不可用
- 用户无法获取翻译后的文章内容
- 对用户体验造成严重影响

---

## 根因分析

### 1. 数据架构设计断裂

**问题描述**：Brief 表和 Articles 表成为数据孤岛，缺乏有效的关联机制。

**具体表现**：
- 两张表独立生成，无外键关联
- Brief 表存储的是渲染后的 HTML 片段
- Articles 表存储原始文章数据
- 前端无法通过 Brief 内容反查 Articles ID

**架构缺陷**：
```
[Brief 表]          [Articles 表]
  - id                - id
  - content_html      - title
  - ...               - content
       ↕ (无关联)        - source_url
                         - ...
```

### 2. 前端依赖 HTML 解析

**问题描述**：前端获取数据的方式依赖解析后端生成的 HTML，而非结构化数据。

**具体表现**：
- 前端无法直接获取数据库 ID
- 使用 `index + 1` 作为代理 ID
- 当数据删除或排序变化时，ID 映射错误

### 3. 分类标签硬编码

**问题描述**：前端分类标签与实际数据源不同步。

**具体表现**：
- 分类标签写死在前端代码中
- 未与后端数据源动态绑定
- 当数据源分类变化时，前端显示过期分类

---

## 修复方案

### 短期修复（已完成）

1. **建立 Brief-Article 关联**
   - 在 Brief 表添加 `article_id` 字段
   - 生成 Brief 时写入对应 Article ID
   - 详情页通过关联 ID 查询数据

2. **修复前端 ID 传递**
   - 将数据库真实 ID 嵌入 HTML data 属性
   - 前端通过 data 属性获取正确 ID
   - 消除 `index + 1` 的错误计算

3. **动态加载分类标签**
   - 从数据源动态获取分类列表
   - 移除硬编码分类标签
   - 确保前后端数据一致

### 长期改进

1. **API 层重构**
   - 引入 RESTful API 或 GraphQL
   - 返回结构化 JSON 数据
   - 减少对 HTML 解析的依赖

2. **数据模型优化**
   ```sql
   -- 建议：添加关联表
   CREATE TABLE brief_articles (
     brief_id INTEGER REFERENCES briefs(id),
     article_id INTEGER REFERENCES articles(id),
     display_order INTEGER,
     PRIMARY KEY (brief_id, article_id)
   );
   ```

3. **前端状态管理**
   - 引入前端状态管理方案
   - 缓存 API 数据
   - 减少重复请求

## 预防措施

### 代码层面

1. **强制数据关联检查**
   - 在 Brief 生成逻辑中添加 Article ID 校验
   - 确保每个 Brief 都有有效的 Article 关联

2. **API 契约测试**
   - 添加前后端数据格式验证
   - 自动化测试覆盖详情页数据加载

3. **禁止硬编码数据**
   - Code Review 中检查硬编码分类标签
   - 分类数据必须从后端获取

### 流程层面

1. **端到端测试**
   - 添加 E2E 测试覆盖关键用户路径
   - 包含分类导航、详情页加载场景

2. **数据完整性监控**
   - 添加 Brief-Article 关联完整性检查
   - 定期扫描孤立数据

3. **发布前验收清单**
   - [ ] 分类标签点击正常显示内容
   - [ ] 详情页正确加载翻译内容
   - [ ] 数据 ID 传递准确

---

## 事故复盘

### 经验教训

1. 数据架构设计应在开发初期确立关联关系，避免后期修补
2. 前端不应依赖解析后端生成的 HTML 获取数据 ID
3. 分类等元数据应从后端动态获取，避免前后端不一致
