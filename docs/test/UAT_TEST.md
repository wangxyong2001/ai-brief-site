## 1. UAT概述

### 1.1 测试目的

验证AI Daily Brief v4系统是否满足用户需求，从用户视角验证功能的可用性、易用性和正确性。

### 1.2 测试角色

| 角色 | 职责 | 参与阶段 |
|------|------|----------|
| 普通用户 | 验证前端浏览体验 | 全部场景 |
| 内容管理员 | 验证简报生成与管理 | 生成管理场景 |
| 技术运维 | 验证监控与告警 | 运维场景 |

### 1.3 测试环境

| 环境 | 地址 | 用途 |
|------|------|------|
| Staging | https://staging.ai-brief.example.com | 功能验收 |
| Production | https://ai-brief.example.com | 上线验收 |

## 2. 用户故事与验收场景

**用户故事**: 作为AI从业者，我希望每天打开网站就能看到最新的AI领域动态简报，以便快速了解行业趋势。

## 6. 缺陷分级标准

| 等级 | 描述 | 示例 | 处理时限 |
|------|------|------|----------|
| P0-致命 | 系统无法使用，核心功能完全失效 | 首页无法加载、生成功能不可用 | 立即修复 |
| P1-严重 | 核心功能受影响但有替代方案 | 某数据源不可用、历史简报无法访问 | 24小时内 |
| P2-一般 | 非核心功能问题，不影响主要流程 | UI样式问题、提示信息不准确 | 1周内 |
| P3-建议 | 优化建议，不影响功能使用 | 字体大小建议调整、文案优化 | 版本迭代时 |

## 9. 附录

### 9.1 测试环境配置清单

```bash
# 环境检查命令
python --version          # Python 3.10+
pip list | grep fastapi   # FastAPI版本
pip list | grep lancedb   # LanceDB版本

# 服务启动命令
uvicorn app.main:app --host 0.0.0.0 --port 8000

# 健康检查命令
curl http://localhost:8000/health
curl http://localhost:8000/ready
curl http://localhost:8000/metrics
```

### 9.2 测试数据准备

```bash
# 生成测试简报
curl -X POST http://localhost:8000/api/brief/generate

# 查询简报
curl http://localhost:8000/api/brief/latest
curl http://localhost:8000/api/brief/list

# 指定日期生成
curl -X POST http://localhost:8000/api/brief/generate \
  -H "Content-Type: application/json" \
  -d '{"date": "2026-05-01"}'
```
