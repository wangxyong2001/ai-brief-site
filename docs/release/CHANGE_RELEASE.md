## 1. 版本管理策略

### 1.1 版本号规范

采用语义化版本控制 (Semantic Versioning)：`MAJOR.MINOR.PATCH`

```
v4.2.1
│ │ │
│ │ └── PATCH: Bug修复、小改动
│ └──── MINOR: 新功能、向后兼容的改进
└────── MAJOR: 重大版本更新、不兼容的变更
```

### 1.2 版本分支策略

```
main (生产分支)
  │
  ├── develop (开发分支)
  │     │
  │     ├── feature/xxx (功能分支)
  │     ├── bugfix/xxx (修复分支)
  │     └── refactor/xxx (重构分支)
  │
  └── hotfix/xxx (紧急修复分支)
```

### 1.3 标签管理

| 标签类型 | 格式 | 示例 | 说明 |
|---------|------|------|------|
| 发布标签 | `v{版本}` | `v4.2.0` | 正式发布版本 |
| 候选标签 | `v{版本}-rc.{n}` | `v4.2.0-rc.1` | 发布候选版本 |
| 测试标签 | `v{版本}-beta.{n}` | `v4.2.0-beta.1` | 内部测试版本 |

### 1.4 版本生命周期

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   开发阶段   │ ──> │   测试阶段   │ ──> │   生产阶段   │
│  (develop)  │     │   (staging) │     │   (main)    │
└─────────────┘     └─────────────┘     └─────────────┘
      │                   │                   │
      v                   v                   v
  持续开发            功能验证             用户使用
  单元测试            集成测试             监控运维
  代码审查            性能测试             问题响应
```

## 3. 变更审批流程

### 3.1 变更分类

| 级别 | 类型 | 示例 | 审批要求 | 通知时间 |
|-----|------|------|---------|---------|
| P0 | 紧急修复 | 安全漏洞、系统崩溃 | 运维主管口头审批 | 立即 |
| P1 | 重大变更 | 架构调整、数据库迁移 | 技术负责人+运维书面审批 | 3天前 |
| P2 | 常规发布 | 新功能、Bug修复 | 技术负责人审批 | 1天前 |
| P3 | 小型改动 | 文案修改、配置调整 | 开发者自审 | 发布当天 |

### 5.2 回滚脚本

#### 5.2.1 快速回滚脚本

```bash
#!/bin/bash
# rollback.sh - 快速回滚脚本
# 用法: bash rollback.sh [版本号]

set -e

# 配置
BACKUP_DIR="/opt/ai-brief/backups"
CURRENT_VERSION=$(cat /opt/ai-brief/.version 2>/dev/null || echo "unknown")
TARGET_VERSION="${1:-}"
LOG_FILE="/var/log/ai-brief/rollback.log"

# 日志函数
log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG_FILE"
}

# 检查参数
if [ -z "$TARGET_VERSION" ]; then
    log "错误: 请指定回滚版本号"
    log "当前版本: $CURRENT_VERSION"
    log "可用备份:"
    ls -la "$BACKUP_DIR" 2>/dev/null || echo "无备份目录"
    exit 1
fi

log "========== 开始回滚 =========="
log "当前版本: $CURRENT_VERSION"
log "目标版本: $TARGET_VERSION"

# 验证备份存在
BACKUP_PATH="$BACKUP_DIR/v$TARGET_VERSION"
if [ ! -d "$BACKUP_PATH" ]; then
    log "错误: 找不到版本 $TARGET_VERSION 的备份"
    exit 1
fi

# 停止当前容器
log "停止当前容器..."
docker-compose down

# 恢复数据
log "恢复数据..."
if [ -d "$BACKUP_PATH/data" ]; then
    cp -r "$BACKUP_PATH/data"/* ./data/
fi

# 加载镜像
log "加载Docker镜像..."
if [ -f "$BACKUP_PATH/ai-brief-image.tar" ]; then
    docker load -i "$BACKUP_PATH/ai-brief-image.tar"
fi

# 启动容器
log "启动容器..."
docker-compose up -d

# 等待健康检查
log "等待容器健康检查..."
sleep 10

# 验证
if curl -f http://localhost:8080/health > /dev/null 2>&1; then
    log "回滚成功! 当前版本: $TARGET_VERSION"
    echo "$TARGET_VERSION" > /opt/ai-brief/.version
else
    log "错误: 健康检查失败，请手动检查"
    exit 1
fi

log "========== 回滚完成 =========="
```

#### 5.2.2 备份脚本

```bash
#!/bin/bash
# backup.sh - 发布前备份脚本
# 用法: bash backup.sh [版本号]

set -e

VERSION="${1:-$(date '+%Y%m%d%H%M%S')}"
BACKUP_DIR="/opt/ai-brief/backups"
BACKUP_PATH="$BACKUP_DIR/v$VERSION"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1"
}

log "========== 开始备份 =========="
log "备份版本: $VERSION"

# 创建备份目录
mkdir -p "$BACKUP_PATH"

# 备份数据
log "备份数据..."
cp -r ./data "$BACKUP_PATH/"

# 备份配置
log "备份配置..."
cp docker-compose.yml "$BACKUP_PATH/"
cp Dockerfile "$BACKUP_PATH/"
cp config.py "$BACKUP_PATH/"

# 导出Docker镜像
log "导出Docker镜像..."
docker save ai-brief-app -o "$BACKUP_PATH/ai-brief-image.tar"

# 记录版本
echo "$VERSION" > /opt/ai-brief/.version

# 清理旧备份（保留最近10个）
log "清理旧备份..."
ls -t "$BACKUP_DIR" | tail -n +11 | xargs -I {} rm -rf "$BACKUP_DIR/{}"

log "备份完成: $BACKUP_PATH"
log "========== 备份完成 =========="
```

### 5.3 回滚场景处理

| 场景 | 症状 | 回滚步骤 | 预计时间 |
|-----|------|---------|---------|
| 启动失败 | 容器无法启动 | 1. 停止容器 2. 恢复镜像 3. 重启 | 2分钟 |
| 健康检查失败 | /health返回错误 | 1. 停止容器 2. 恢复镜像和数据 3. 重启 | 5分钟 |
| 数据迁移失败 | 数据库错误 | 1. 停止容器 2. 恢复数据 3. 重启 | 10分钟 |
| 功能异常 | 用户报告问题 | 1. 评估影响 2. 执行完整回滚 | 15分钟 |
| 性能下降 | 响应时间增加 | 1. 分析原因 2. 必要时回滚 | 30分钟 |
